Under the hood

32 internal playbooks, in plain English

A "skill" is a step-by-step playbook the team quietly follows — how a mock-up gets approved, how content gets a rights check, how the cheapest AI model gets picked. You never load or run these yourself; they're here purely for anyone curious about how the reliability above is actually achieved.

The coordinator

studio
The master playbook — how the Project Lead runs the whole nine-stage journey, assigns your project's size, and brings in specialists.

Staying on track over a long build

focus-guard
Keeps a tiny, always-current note of the one active goal, so a long project never quietly drifts off course — even across many separate sessions.
quality-gate
A checklist nothing can skip before a stage is backed up or published — tests, an independent review, security and licence checks, accessibility, and up-to-date docs.
audit-loop
The systematic protocol for any "keep checking until it's clean" request — plans the full coverage upfront rather than adding one check at a time.
yagni-rules
The lean-coding rule the whole team follows: build only what's genuinely needed, nothing "just in case."
cost-guard
The confirmed cheapest-option-first spending default, and the plan for a long build that might span several days.
model-router
Automatically picks the right AI model and effort for each individual step, spending more only where it genuinely matters.

Remembering everything between sessions

dev-memory
The plain-text notebook system that makes a project resumable — in the same session, or a brand-new one, days later.
memory-graph
A compact index and a small "map" of how everything connects, so recalling a big project costs far less and finds just what's relevant.

Planning, mock-ups, and building in stages

warframe-prototype
The rule that a clickable mock-up must be shown and approved before any real code is written.
phased-roadmap
Turns the approved design into "smallest useful version first, then improvements in order."
micro-task-planning
Breaks the plan into small, independently checkable steps, each with one clear way to prove it's done.
tdd-workflow
On bigger projects, makes sure a test that must fail is written before the matching code exists — proof-first, not proof-after.

The command centre

command-centre
Defines pause, resume, stop, skip, and schedule-for-later, plus the one-page dashboard showing your app's concept, design, plan and content.

Real content, real AI, always with your consent

content-creation
The whole Content stage: planning your app's real text and media, and the manifest that tracks each asset's approval and rights.
gemini-integration
The opt-in protocol for generating images, audio and video with your own Google account — always a cost estimate and a yes first.
ollama-integration
The protocol for using a free, local AI model instead of the cloud — always your choice, always asking first.

Shipping safely

publish-github
How a finished project is published privately to your own GitHub, with a separate, explicit step to make it public.
checkpoint-commit
The private backup taken after each build stage, so nothing is lost even before you're ready to publish.

Fixing things quietly

self-healing
Lets the Fixer quietly try a couple of small repairs on its own first, and only interrupt you if that doesn't work.

Getting started & finding existing tools

first-run
The one-off "getting to know you" setup that runs before your very first project, never again after.
ecosystem-finder
Suggests an existing Claude Code tool when one already solves a specific need — always asking before installing anything.
