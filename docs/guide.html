Getting started

From nothing installed to your first published app

This is the same journey described in the project's README, laid out as a step-by-step guide. Nothing here needs technical knowledge — if you can describe your idea to a friend, you can follow this.

1. Install Claude Code

GRU953-Studio is an add-on for Claude Code, Anthropic's coding assistant that you talk to in plain English. It is not a separate program, and it does not run inside the Claude Desktop chat app — it needs Claude Code specifically. It works both in the desktop app and on Claude Code on the web, from a browser.

Install and sign in to Claude Code first, then come back here.

2. Add GRU953-Studio

Type these two lines into Claude Code, one at a time, pressing Enter after each:

/plugin marketplace add GRU-953/GRU953-Studio
/plugin install gru953-studio@gru953-studio

Prefer a downloaded file instead? Open the Releases page, download the newest .zip, unzip it anywhere, then run the two lines above but replace GRU-953/GRU953-Studio with the folder path you unzipped to.

3. Two free things it relies on

Node.js

A small free program that powers the built-in safety checks. Install it once, before you first publish.

GitHub CLI

Only needed when you publish. After installing, run gh auth login once and follow the prompts.

If either is missing exactly when it's needed, GRU953-Studio stops and tells you precisely what to install — it never fails silently.

Optional, only for generated images/audio/video: a Google Gemini API key of your own. Entirely optional, off unless you turn it on, and the studio always shows you the cost and asks first. Written text needs none of this.

4. The first time you use it

It runs a short, one-off "getting to know you" setup — your preferred name, the kinds of things you like to build, your GitHub username. This never happens again.

After that, every project follows the same loop:

Describe it

Type your idea between square brackets, e.g. [ a one-page website listing my favourite books ].

Answer the pop-ups

Simple multiple-choice questions, one at a time, each with a recommended answer marked.

Approve the mock-up

A clickable preview of the screens, plus what ships first — approve both before real code exists.

It writes the content

Real Bangla/English text; media only with your yes and a shown cost.

It builds and tests it

In stages, checking in at every meaningful point.

You review, then it publishes

Private to your own GitHub by default; public is a separate step you choose.

5. Staying in control while it works

You never have to sit and watch. A few simple commands, typed any time:

CommandWhat it does
/studio-statusA plain-English progress report — done, in progress, next, and anything waiting on you.
/studio-pausePause the current task. Nothing is lost.
/studio-resumePick up exactly where you paused.
/studio-stopSet everything down cleanly for the day.
/studio-skipSet the current task aside and move to the next one; nothing is deleted.
/studio-scheduleAsk it to pick a task back up at a time you choose.
/studio-dashboardOpen a one-page visual summary of your app's concept, design, plan and content.
