A free plugin for Claude Code

It's built, filled with real content, and shipped.

GRU953-Studio is a team of AI specialists you talk to like a person — no coding, no jargon. It designs your app, shows you a clickable mock-up first, writes its real text in Bangla and English, builds it natively for whichever platform you use, and publishes it to your own GitHub — only when you say so.

Get started See what people build

Install inside Claude Code

/plugin marketplace add GRU-953/GRU953-Studio
/plugin install gru953-studio@gru953-studio
What you get

One coordinator. A whole studio behind them.

You only ever talk to the Project Lead. Everything below happens quietly, only when your particular project actually needs it.

See it before it's built

A clickable mock-up of your app, plus a plain-English plan for what ships first — you approve both before real code exists.

Real content, not placeholders

Buttons, onboarding and messages written natively in Bangla and English — plus optional AI-generated images, audio and video, with your yes first.

Every platform, one team

Android, iPhone/iPad, Mac, Windows and Linux — with a dedicated specialist for each of 10 languages, so the right expert builds the right platform.

Stay in control

Pause, resume, stop, skip, or schedule any piece of work — and a one-page dashboard shows your app's concept, design, plan and content at a glance.

Never loses your place

Everything is saved in plain-text notes. Close your laptop, come back next week or open it on Claude Code on the web — it resumes exactly where it stopped.

Cheapest option first

An automatic chooser picks the right AI model and effort for each step, spending more only where it genuinely matters — and always asks before anything unusual.

Grows in safe stages

The smallest useful version first, then improvements in order — with a private backup of your work after every stage, so nothing is ever lost.

Safety built in

Automatic checks for leaked passwords, known weaknesses and licence problems — publishing is private by default, public only as a separate, explicit step.

How it works

Eight steps. All of them plain English.

Tell it your idea

Type it between square brackets, or just type /studio.

Answer a short interview

Only as many pop-up questions as it genuinely needs, each with a recommended answer marked.

It sizes the job

Tiny, Standard, or Complex — in plain words, with a reason. You can raise or lower it any time.

You see it before it's built

A clickable mock-up and a phased plan — nothing real is built until you approve both.

It writes the real content

Bangla and English text; generated media only with your yes and a shown cost.

It builds, checks, and tests

In stages, with a private backup after each one. Stuck points are explained plainly.

It publishes — only when you say so

Private by default, to your own GitHub account. Public is a separate, later step.

It remembers everything

Pause any time. Come back whenever. It picks up exactly where it stopped.

Built openly

Nothing about how this works is hidden.
