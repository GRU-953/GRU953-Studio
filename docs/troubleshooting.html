Troubleshooting

Plain fixes for common bumps

GRU953-Studio is built to explain problems in plain English rather than show scary error text. If something below doesn't match what you're seeing, open an issue on the GitHub issues page — describe what you did and what happened, no technical detail required.

What you see or noticeWhat it meansWhat to do
command not found: claudeClaude Code isn't installed yetInstall Claude Code first, then try again
It won't work in the Claude Desktop appGRU953-Studio needs Claude Code, a different (though related) toolUse Claude Code, not the Desktop chat app
/plugin install didn't seem to take effectThe plugin list sometimes needs a refreshClose and reopen Claude Code, then try the install line again
A message about Node.js or a missing safety checkNode.js isn't installedInstall Node.js (free), then retry
It says it can't publish / mentions gh or "not signed in"The GitHub tool isn't set upInstall the GitHub CLI, run gh auth login, follow the prompts
It stopped part-way (e.g. you ran out of daily usage)Nothing is lost — it saves progress constantlyCome back later and say "continue" or run /studio again; it resumes exactly where it stopped
It refused to publish unexpectedlyA safety check did its job (an unconfirmed publish, or a password spotted in the files)Read the plain-English reason it gives, fix that one thing, and try again — this is protection, not a fault
It asked before generating an image/audio/videoThe Gemini cost/privacy check is working as designedSay yes if you're happy with the cost, or ask it to try a different description
It says Gemini isn't available hereNo Google account/key is set up, or the session can't reach the internetFollow the step-by-step guide it gives you, or skip generated media — written content needs none of it
A pop-up used a word you didn't knowIt should explain every term onceAsk it "what does that mean?" in plain words — it's happy to explain
It says your project is bigger than it should take onAn honest limit, not a failure — very large, complex systems aren't what this tool is forAsk for the smallest useful first version instead, and grow it from there in stages
A generated illustration or clip isn't quite rightGenerated media is a first attempt, not guaranteed perfectAsk for it to be regenerated with a different description — you're asked before each attempt, so nothing repeats without your yes
You're not sure what stage your project is atEasy to lose track over a long buildRun /studio-status for a plain-English report, or /studio-dashboard for the full visual picture

Still stuck? Open an issue on the GitHub issues page describing what you did and what happened. You don't need to know any technical terms — just describe it the way you would to a friend.
